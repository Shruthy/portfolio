ARTECA Usability Study

ARTECA is a platform for experimental publishing and a curated space for essential content linking the arts. science and technologies.

My Goals

Drive usability test
Create tasks for usability tests
Identify problems users face with ARTECA website
Recommend and prototype new designs to make the website user centric
Use Google Analytics to study conversion rate and track user behavior

Planning

For every usability test we have a goal defined. I identify a set of tasks for users. These tasks help us get some insights about users experience with the website.

Contemplate users behavior

During the first usability study I proctored, a participant did not notice a block of content that was visibly placed on the webpage. I did not ask any follow-up questions during the test and that was a big mistake which only left me with assumptions as to why they did not take action on the block of content.

I learned to ask participants unbiased questions on the go about their choices. For instance I ask participant to find a journal issue on ARTECA website. If they prefer to search first instead of browsing - it could mean that browsing is difficult or maybe time consuming. Instead of making the assumption I ask about their choices. From this experience I learned to be curious and ask follow-up questions.

Analyzing data

Observations and Insights

  • Participants felt the website was simple and easy to use
  • "I like the yellow bar" - The yellow bar attracts users attention.
  • Most participants first explored on the top right region of navigation bar to find sign up option - This tells us about the users mental model
  • Some participants have hard time noticing the top navigation bar - "Oh I did not see that (top navigation bar)" Participants preferred to use search feature over browsing through the catalog

Face Reader

I studied the participants emotions using Noldus face reader software. This helped me understood what emotions participants felt while interacting with the website.
